# 🛡 云片滑块验证码逆向实战

**声明：** 本项目及文档仅供网络安全技术交流与学习，严禁用于任何非法用途及商业接口滥用。

---

## 📖 案例背景

云片滑块是业界著名的行为风控产品，集成了指纹收集、AES/RSA 混合加密、JSONP 协议混淆、以及基于机器学习的轨迹状态机检测。

---

## 🧠 核心逆向历程与技术复盘

### Phase 1: 协议欺骗与破局定位 (从 XHR 陷阱到 Hook 降维打击)

**踩坑记录：**  
起初按常规思路尝试 XHR 断点抓取加密入口，却发现根本断不住。

**破局反思：**  
意识到目标接口 `/v1/jsonp/captcha/get` 使用了 JSONP 协议（通过动态创建 `<script>` 标签发起请求，绕过跨域限制），传统的 XHR/Fetch 断点对其完全失效。

**终极定位：**  
放弃追溯网络请求，转而利用 **子树修改断点** 和 **`JSON.stringify` Hook**。在 Call Stack (调用栈) 中配合 Scope (作用域) 追踪，成功在加密发生的“前一秒”拦截到明文环境指纹（`browserInfo`、`fp` 等）。

---

### Phase 2: 密码学还原 (AES-CBC + RSA 混合)

**加密架构剖析：**

- **`i` 参数**：明文数据经过随机生成的 **16位 Key** 和 **IV**，使用 **AES-CBC** 模式加密。
- **`k` 参数**：将上述 AES 的 Key 和 IV 拼接，使用服务端的 **RSA 公钥**加密。

**工程实现：**  
使用 Python 的 `pycryptodome` 库重写加密逻辑，成功避开 CryptoJS 默认的隐式转换机制，稳定获取到第一阶段的 `token` 和图片 URL。

---

### Phase 3: 视觉对抗与状态机映射 (OCR 翻车与 OpenCV 救场)

**踩坑记录：**  
在计算滑块缺口距离时，过度依赖免费的 `ddddocr`。由于云片滑块图自带 Alpha 透明通道，且背景图左侧有深色阴影干扰，导致 `ddddocr` 频繁识别出极其离谱的错误距离。这导致在后续测试中，一直误以为是“轨迹模拟算法”写错了。

**破局反思：**  
果断弃用普通 OCR。引入 **OpenCV 的 Canny 边缘检测算法**。通过转化为灰度图并提取轮廓特征，彻底无视了颜色、阴影和透明度的伪装，精准锁定真实的缺口像素距离。

**状态机几何换算：**  
识破了云片的“视觉欺骗”。

- `width: 480`（原图宽），`winWidth: 330`（网页渲染宽）。
- 提交的比例 `distanceX = 物理识别距离 / 480`。
- 轨迹所需的绝对位移 `actual_move = 物理识别距离 * (330 / 480)`。

---

### Phase 4: 轨迹伪造与 AI 协作博弈 (最深刻的教训)

**AI 的砒霜 (双重编码惨案)：**  
在拼装最终的 Verify 请求时，由于听信了 AI 提供的防御性代码 `urllib.parse.quote(i_ver)`，结合 `requests.get(params=...)` 的底层机制，导致了致命的**双重 URL 编码**。密文变成乱码，服务器持续返回 `param_invalid` (参数无效)。

**独立思考的胜利：**  
通过严密的逻辑复盘，最终排除了 AI 的误导代码，成功将报错推进到 `verify fail`（参数合法，但轨迹被风控拦截）。

**降维打击 (Data Replay)：**  
采用最高维度的 **“真人轨迹等比缩放 (Scale Real Track)”**。通过 Hook 拿到纯原味的真人野生轨迹，保留其 Y 轴抖动和时间戳特征，仅对 X 轴距离进行动态缩放。彻底骗过 AI 风控，达成 `code: 0, result: true`。

## 🚀 核心技术栈

- **语言**: Python 3.x / JavaScript
- **逆向工具**: Chrome DevTools, Hook 脚本
- **密码学**: `pycryptodome` (AES, PKCS1_v1_5, Base64)
- **图像识别**: `opencv-python` (`cv2.Canny`, `cv2.matchTemplate`)
- **网络通信**: `requests`, `urllib`, `json`, `re`

## 心得

> **“不要陷入局部最优解的死胡同。”**  
> 当轨迹怎么调都不对时，可能问题根本不在轨迹，而是在最开始的图像识别（距离算错了）或者传输协议（URL 编码多此一举）上。

> **“AI 是副驾，但方向盘必须在自己手里。”**  
> AI 可以极大地提升编写加解密算法和 OpenCV 的效率，但其提供的代码必须经过工程师的独立思考与沙盘推演。缺乏底层的网络协议常识，很容易被 AI 的幻觉代码带进沟里。

> **“用魔法打败魔法。”**  
> 在对抗最顶级的行为风控模型时，人类真实产生的数据（Data Replay）永远比再精妙的数学物理公式都要无懈可击。
