# 📝 控制流逆平坦化 (Unflattening Control Flow)

## 1. 核心理论概念
- **原理简述**：控制流平坦化（Control Flow Flattening）是一种破坏代码原本从上至下执行顺序的技术。它将顺序执行的代码块打碎，放入一个巨大的 `switch-case` 结构中，并由一个外部的 `while(true)` 循环和一个状态机数组（如 `'3|1|2'.split('|')`）来动态决定执行路径。 
- **应用场景**：应对高级反爬虫手段（源自 OLLVM 等底层编译器保护的思想），解决“业务逻辑变成了面条迷宫，彻底失去可读性”的问题。

## 2. 逆向分析切入点
- **特征识别**：非常明显！代码中出现大量的 `while (!![])` 或 `while (true)`，里面紧跟着一个巨大的 `switch (变量)`。而在循环外部，一定有一个形如 `var _0xabc = '4|1|3|0|2'.split('|')` 的分发器。
- **定位技巧**：在 AST 中定位 `WhileStatement`，检查其子节点是否只包含一个 `SwitchStatement`。同时向上层作用域寻找那个控制流数组的 `VariableDeclarator`。

## 3. 核心突破逻辑
- **步骤一：提取分发器**：分析 AST 树，找到那个分发字符串（如 `'4|1|3|0|2'`），并在本地 `split` 成真实的执行顺序数组 `[4, 1, 3, 0, 2]`。
- **步骤二：抓取代码块**：遍历内部的 `switch-case` 节点，将每一个 `case` 内部真实的业务代码（`BlockStatement`）提取出来存入字典，以 `case` 的数字作为 Key。
- **步骤三：有序拼接**：根据步骤一提取的顺序数组，从步骤二的字典中依次把代码块拿出来，平铺成一个正常的顺序结构。
- **步骤四：清理外壳**：用拼接好的正常顺序代码块，替换掉那个毫无用处的 `while-switch` 父节点，完成逆向平坦化。

## 4. 易错点与踩坑记录
- ❌ **错误尝试**：在提取 `case` 代码块时，连同混淆器自动生成的 `continue;` 或 `break;` 一起提取进去了，导致重组后的代码在运行到一半时莫名其妙跳出执行流。
- ✅ **正确思路**：在处理每个 `case` 的 AST 节点时，必须手动滤除掉用来维持 `switch` 运转的尾部 `ContinueStatement` 或 `BreakStatement`。
