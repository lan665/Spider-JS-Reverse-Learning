# 💻 实战案例复盘：企业级风控 Cookie 参数逆向工程

## 1. 目标分析

**目标参数**：`TYCID` （存储于 Cookie，用于 Web 端的身份校验与风控追踪）。

**防护特征**：高强度 OB 混淆（Obfuscator）、无限 `debugger` 反调试陷阱、`document.cookie` 原生对象劫持、动态滑块验证。

## 2. 第一阶段：参数定位过程复盘（动态追踪路径）

### 步骤 1：发包拦截器分析（初次尝试）

**动作**：通过 XHR 断点或拦截器（`request.use`）断住请求。

**发现**：发现 `X-Tycid` 或请求头中的相关参数只是简单地从 `document.cookie` 中提取，并没有当场加密。

**结论**：该参数属于“提前生成型”，真正的算法在页面加载早期的风控脚本中。

### 步骤 2：Hook 写入点与反调试突破

**动作**：清空 Cookie 尝试 Hook `document.cookie`。

**困境**：触发了“无限 `debugger`”和滑块验证，导致执行流无法到达 Hook 点。

**破局**：在 Chrome 中对反调试 VM 文件设置“一律不在此处暂停（Never pause here）”，强行跳过环境体检陷阱。

### 步骤 3：识破对象劫持

**关键发现**：通过分析调用栈发现，网站并未使用简单的赋值，而是通过 `Object.defineProperty` 重写了 `document.cookie` 的 `set` 方法。

**定位结果**：锁定了一个名为 `_0x5e8995`（名称随版本变化）的 Getter 函数，它是 `TYCID` 生成的“最终加工厂”。

## 3. 第二阶段：加密算法还原过程复盘（静态分析路径）

### 步骤 1：OB 混淆核心部件提取

**动作**：从源码中剥离“解密三部曲”：

- **部件 A（大数组）**：包含几千个 Base64 编码的乱码字符串。
- **部件 B（自执行位移函数）**：负责将数组旋转到正确顺序。
- **部件 C（解密函数）**：核心调用接口，内部含 RC4 解密逻辑。

**修复**：在本地 Node.js 环境中删除了解密函数内的“防格式化（Regex）”自卫陷阱。

### 步骤 2：AST 自动化清洗（Babel 插件）

为了解决代码中几万处乱码调用，编写了 Babel 脚本进行降维打击：

- **常量折叠**：将 `"add" + "E" + "vent"` 自动合并为 `"addEventListener"`。
- **解密调用还原**：遍历语法树，将 `_0x4649(0x123, '...')` 替换为真实的明文字符串。
- **马甲函数复活**：通过 `eval` 动态复活多参数包装函数，实现递归还原。

### 步骤 3：逻辑剥离与算法定位

**清洗结果**：生成 `decode_source.js`，原本的天书代码变回了可读的逻辑。

**锁定算法**：

- 在清洗后的代码中搜索 `Object.defineProperty` 或 `.cookie`。
- 发现算法输入依赖于 `window.Date`（时间戳）和一系列环境指纹采集。

**最终形态**：通过本地 Node.js 补齐环境（补 `navigator`、`location` 等），成功在本地生成了合法的 `TYCID` 字符串。

## 4. 经验总结与复盘启示

- **静态胜过动态**：当动态调试被无限 `debugger` 卡死时，通过 AST 清洗源码（静态分析）是唯一的降维打击手段。
- **环境体检是核心**：`TYCID` 不仅仅是一个 MD5，它包含了 WAF 对你浏览器环境的深度校验（鼠标轨迹、UA、屏幕分辨率）。
- **逆向策略**：面对此类天花板级别对抗，“补环境黑盒调用” 的性价比远高于 “纯算法逻辑扣取”。
