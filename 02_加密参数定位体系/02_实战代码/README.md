## 🚀 核心案例 1：U-Sign 签名生成 (sign.py)

> 📖 **深度剖析：** 详细的逆向定位过程与脱敏图文分析，请参阅 [学习笔记 - U-Sign 请求头签名](../01_学习笔记/sign_请求头签名_发包层/README.md)

### 1. 原理概述
该脚本还原了目标网站生成防篡改签名 (U-Sign) 的核心逻辑。原前端加密函数同时接收请求路径 (URL) 和请求体 (Payload) 作为入参，内部写死了特定盐值 (Salt)。本脚本主要针对目标接口的特性，完整复现了其无 URL 查询参数时的 POST 请求加密分支。
经逆向分析确认，目标采用**标准 MD5** 算法（无魔改）。
* **明文拼接规则：** `无空格格式化的 JSON Payload + "&" + 盐值`
* **处理动作：** 全局转为小写后，进行 MD5 摘要计算。

> **⚠️ 合规声明：** 遵循网络安全合规要求，脚本中硬编码的真实 `Salt` 已进行脱敏替换。本目录下的代码仅供逆向工程技术原理交流与面试作品展示，严禁用于任何商业目的或恶意的自动化高频抓取。

### 2. 环境依赖
* Python 3.6+
* 纯原生实现，仅依赖 Python 内置库 `hashlib` 和 `json`，无需额外安装第三方包。

### 3. 使用方法
在终端中进入本目录，直接运行该脚本，即可输出模拟数据生成的签名结果：

```bash
python sign.py
```

## 🚀 核心案例 2：响应体 AES 解密 (response.py)

> 📖 **深度剖析：** 包含 Hook `JSON.parse` 拦截与 Call Stack 溯源的完整教学，请参阅 [学习笔记 - 响应体解密拦截](../01_学习笔记/response_响应体解密_接收层/README.md)

### 1. 原理概述
该脚本还原了目标网站在 Axios 响应拦截器中对 `response.Data` 密文进行解密的核心逻辑。我们通过 Hook `JSON.parse` 成功拦截明文，并通过调用栈回溯突破闭包作用域，提取出了硬编码的 Key 与 IV。
经逆向分析确认，目标采用标准 **AES-CBC** 对称加密算法，并使用 PKCS#7 填充。
* **Key (密钥)：** 前端硬编码的 32 位 Hex 字符串（已脱敏）。
* **IV (偏移量)：** 前端硬编码的 16 位 Hex 字符串（已脱敏）。

### 2. 环境依赖
* Python 3.6+
* 依赖第三方密码学库：`pip install pycryptodome` (注意：代码中导入包名需写为 `Crypto`)

### 3. 使用方法
在终端中进入本目录，直接运行该脚本。脚本将对内置的测试 Base64 密文进行解密，并在终端打印结构化的 JSON 明文：

```bash
python response.py
```
