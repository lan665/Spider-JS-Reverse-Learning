## 🚀 核心案例：U-Sign 签名生成 (sign.py)

### 1. 原理概述
该脚本还原了目标网站生成防篡改签名 (U-Sign) 的核心逻辑。原前端加密函数同时接收请求路径 (URL) 和 请求体 (Payload) 作为入参，内部写死了特定盐值 (Salt)。本脚本主要针对目标接口的特性，完整复现了其无 URL 查询参数时的 POST 请求加密分支。
经逆向分析确认，目标采用**标准 MD5** 算法（无魔改）。
* **明文拼接规则：** `无空格格式化的 JSON Payload + "&" + 盐值`
* **处理动作：** 全局转为小写后，进行 MD5 摘要计算。

> **⚠️ 合规声明：** 遵循网络安全合规要求，脚本中硬编码的真实 `Salt` 已进行脱敏替换。本目录下的代码仅供逆向工程技术原理交流与面试作品展示，严禁用于任何商业目的或恶意的自动化高频抓取。

### 2. 环境依赖
* Python 3.6+
* 纯原生实现，仅依赖 Python 内置库 `hashlib` 和 `json`，无需额外安装第三方包。

### 3. 使用方法
在终端中直接运行该脚本，即可输出模拟数据生成的签名结果：

```bash
python sign.py
