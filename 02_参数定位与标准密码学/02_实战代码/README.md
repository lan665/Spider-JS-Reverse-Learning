## 🚀 核心案例 1：U-Sign 签名生成 

> 📖 **深度剖析：** 详细的逆向定位过程与脱敏图文分析，请参阅 [学习笔记 - U-Sign 请求头签名](../01_学习笔记/sign_请求头签名_发包层/README.md)

### 1. 原理概述
该脚本还原了目标网站生成防篡改签名 (U-Sign) 的核心逻辑。原前端加密函数同时接收请求路径 (URL) 和请求体 (Payload) 作为入参，内部写死了特定盐值 (Salt)。本脚本主要针对目标接口的特性，完整复现了其无 URL 查询参数时的 POST 请求加密分支。
经逆向分析确认，目标采用**标准 MD5** 算法（无魔改）。
* **明文拼接规则：** `无空格格式化的 JSON Payload + "&" + 盐值`
* **处理动作：** 全局转为小写后，进行 MD5 摘要计算。

### 3. 使用方法

## 🚀 核心案例 3：动态 Header 签名与作用域穿透对抗

> 📖 **深度剖析：** 包含突破 Webpack 混淆、作用域跳跃与“夹逼断点”定位加密源头的完整实战记录，请参阅 [学习笔记 - 突破底层调度栈与动态 Header 签名](../01_学习笔记/case2_sign/README.md)

### 1. 原理概述
该脚本还原了某招标数据平台（标找找/企查查体系）在发包前动态生成加密请求头 `X-Kzz-Request-Key` 的核心逻辑。
在逆向过程中，成功克服了 Webpack 代码重度压缩引起的“断点漂移”，以及 Axios 底层 Promise 异步调度造成的“调用栈断层”问题，通过 DevTools 的 Scope（作用域）面板精准“空降”至真实的业务请求拦截器。

经算法特征定性与控制台黑盒验证，确认采用**标准 MD5** 算法。
* **明文拼接规则：** `固定盐值(Salt) + ":" + 平台标识(from) + ":" + 13位时间戳(time) + ":" + UUID格式随机数(id)`
* **处理动作：** 拼接后进行 MD5 摘要计算，并统一转换为 32 位大写十六进制字符串。
* **进阶风控发现：** 接口除校验 Header 签名外，还结合了深度的业务授权与环境指纹校验。无 Cookie 访问会触发 `429 Too Many Requests`，非标准会员拉取超量数据会触发 `403 Limitation`。
